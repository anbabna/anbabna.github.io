GitHub部署的Chirpy博客Footer配置

专为GitHub Pages优化的footer.html配置方案

⚠️ GitHub Pages特别注意事项

GitHub Pages对JavaScript有限制，需要特别注意：

使用async属性加载不蒜子脚本，避免阻塞页面
确保所有外部资源使用HTTPS协议
GitHub Pages会自动构建Jekyll站点，无需本地构建

创建footer.html文件

在您的Chirpy博客项目中，创建文件：_includes/footer.html

操作步骤：

在GitHub仓库中创建_includes文件夹
在_includes文件夹中创建footer.html文件
将以下代码复制到文件中
提交更改到GitHub，等待自动部署

# 站点基本设置 title: "您的博客标题" url: "https://您的用户名.github.io" baseurl: "{{ site.baseurl }}" timezone: "Asia/Shanghai" # 作者信息 author: name: "您的姓名" email: "您的邮箱" github: "您的GitHub用户名" # 不蒜子统计配置 busuanzi: enable: true site_uv: true site_pv: true page_pv: true # 社交链接 social: links: - url: "https://github.com/您的用户名" icon: "fab fa-github" title: "GitHub" - url: "mailto:您的邮箱" icon: "fas fa-envelope" title: "Email" # GitHub Pages构建设置 remote_theme: "cotes2020/jekyll-theme-chirpy" plugins: - "jekyll-paginate" - "jekyll-sitemap" - "jekyll-seo-tag" - "jekyll-feed"

GitHub工作流程

1. 在GitHub仓库中创建或更新文件
2. 提交更改到main分支
3. GitHub Actions会自动构建和部署
4. 访问 https://您的用户名.github.io 查看效果
5. 如果使用自定义域名，需要在仓库Settings中配置
